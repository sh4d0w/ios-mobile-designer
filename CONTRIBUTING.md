# Contributing to iOS Mobile Designer

Thank you for your interest in contributing to iOS Mobile Designer! This document provides guidelines and steps for contributing.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue with:
- A clear, descriptive title
- Steps to reproduce the issue
- Expected vs actual behavior
- Framework (SwiftUI/React Native/Flutter) if relevant
- Any error messages or screenshots

### Suggesting Enhancements

We welcome suggestions for:
- New iOS components
- Additional framework support
- Improved accessibility guidelines
- Better code examples
- Documentation improvements

Please open an issue describing:
- The enhancement you'd like to see
- Why it would be useful
- Any implementation ideas you have

### Pull Requests

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create a branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make your changes** following our style guide
5. **Test** your changes
6. **Commit** with a clear message:
   ```bash
   git commit -m "Add: new component for X"
   ```
7. **Push** to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
8. **Open a Pull Request** against the `main` branch

## Style Guide

### SKILL.md Contributions

When adding or modifying skill content:

1. **Follow the existing structure**:
   - Each section should cover SwiftUI, React Native, and Flutter
   - Include working code examples
   - Document accessibility considerations

2. **Code examples should**:
   - Be production-ready and tested
   - Follow Apple HIG specifications
   - Include comments explaining key decisions
   - Use semantic colors and Dynamic Type

3. **Formatting**:
   - Use proper Markdown syntax
   - Include code blocks with language identifiers
   - Use tables for specifications and comparisons

### Commit Messages

Use clear, descriptive commit messages:
- `Add: new component or feature`
- `Fix: bug description`
- `Update: improved existing feature`
- `Docs: documentation changes`
- `Refactor: code improvements without feature changes`

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on the content, not the person
- Help others learn and grow

## Questions?

If you have questions about contributing, feel free to:
- Open an issue with the `question` label
- Start a discussion in the repository

Thank you for helping make iOS Mobile Designer better!
